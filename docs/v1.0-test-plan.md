# Test Plan
Generated by /plan-eng-review on 2026-05-19 (revised — FalkorDB Lite, no daemon)
Branch: main
Repo: etcircle/cga (CodeGraphAgent / CGA) — new repo, **v1.0 milestone**

Scope: v1.0 = FalkorDB Lite (transplanted) + transplanted engine + rewritten
query core + MCP client + 3 tools + `index_status`. Proven on a small Python
repo first, then di-copilot as a scale test. No `cga` daemon (FalkorDB Lite's
worker is the server). No CLI client, no web UI in v1.0.

## Key Interactions to Verify
- MCP tool call → query core (in-process) → FalkorDB Lite socket → result, for
  each v1.0 tool.
- Cold index (async): the first call on an unindexed repo returns
  `{status:indexing}`; `graph_builder` runs in a background thread; the agent
  polls `index_status` until `indexed`, then re-issues. Incremental re-index of
  a few changed files (mtime scan) is synchronous inline.
- FalkorDB Lite worker auto-spawns on first connection; a second `cga` client
  process connects to the same worker concurrently with no error.
- A file edit → next query reflects it via the mtime scan (staleness window =
  1 query).
- `index_status` before a query → correct indexed / fresh / indexing state.

## Edge Cases
- Cold index of a large repo (di-copilot scale test) → async, never blocks.
- Two `cga` client processes start together → both ensure the FalkorDB worker;
  the loser's worker fails to bind the socket and exits cleanly; no corruption.
- Query miss → `{results:[],status:empty}`; ambiguous → candidates +
  `status:ambiguous`; unparseable file → skipped, surfaced in `warnings`.
- Config severing → an engine module imported with no `cli/` dependency.
- Transplanted parser on a file the fork handled → identical graph nodes/edges
  vs current CGC output.
- FalkorDB worker fails to spawn → MCP client surfaces a clear error, not silent.

## Critical Paths
- The small proving repo indexes cold and all v1.0 tools answer correctly,
  hitting the D8 numeric bars.
- An agent (MCP client) completes a real query end-to-end with the FalkorDB
  worker auto-spawned — no manual setup.
- After an edit, a query returns fresh results with no manual re-index.
